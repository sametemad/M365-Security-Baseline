Conditional Access

Objective

Implement Conditional Access policies to strengthen Microsoft 365 identity security and reduce unauthorized access risks.

Environment

* Microsoft Entra ID
* Microsoft 365 Business
* Exchange Online
* SharePoint Online
* Teams

Security Goals

* Protect privileged accounts
* Require MFA for remote access
* Block legacy authentication
* Reduce account compromise risks

Policies Implemented

Administrative Accounts

Policy:

* Require MFA
* Restrict high-risk sign-ins
* Monitor privileged access

Remote Access

Policy:

* Require MFA for external connections
* Validate user identity before access

Legacy Authentication Blocking

Policy:

* Block POP3 authentication
* Block IMAP authentication
* Block SMTP basic authentication where applicable

Risk-Based Protection

Policy:

* Monitor suspicious sign-in activity
* Review risky users
* Investigate unusual login behavior

Validation

Verified:

* MFA enforcement
* Successful user access
* Blocked unauthorized sign-ins
* Policy application logs

Benefits

* Improved identity protection
* Reduced phishing exposure
* Stronger access control
* Enhanced Microsoft 365 security posture
